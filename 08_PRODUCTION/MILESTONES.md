# MILESTONES

## Purpose

This document defines the major development milestones for ROOTS.

A milestone represents a significant project achievement. Every completed milestone should result in a playable, testable version of the game.

---

# Development Philosophy

ROOTS should be built incrementally.

Each milestone must improve the game while keeping the project stable.

Never begin a new milestone until the previous one is complete and verified.

---

# Milestone 1 — Project Foundation

## Goal

Create a clean and scalable Unity project.

### Tasks

* Create Unity project
* Configure project settings
* Create folder structure
* Configure Git repository
* Import required packages
* Configure Input System
* Configure URP (if used)
* Configure coding standards
* Create base scenes
* Verify project builds successfully

### Deliverable

A clean Unity project ready for development.

---

# Milestone 2 — Core Gameplay Systems

## Goal

Implement every fundamental gameplay system.

### Systems

* First Person Controller
* Camera
* Interaction System
* Inventory
* Evidence System
* Dialogue System
* Objective System
* Save System
* Door System
* Basic UI

### Deliverable

A fully playable gameplay prototype.

---

# Milestone 3 — World Prototype

## Goal

Create the first playable environment.

### Content

* Small neighborhood
* One playable house
* Basic interiors
* Lighting
* Environment props
* Ambient audio
* Navigation

### Deliverable

The player can freely explore a small section of the world.

---

# Milestone 4 — Vertical Slice

## Goal

Create the first complete gameplay experience.

### Includes

* Opening sequence
* First investigation
* NPC interactions
* Evidence collection
* Puzzle
* Story progression
* Save/Load
* Final objective

### Deliverable

A polished 15–20 minute gameplay demo representing the final quality target.

---

# Milestone 5 — Full Production

## Goal

Build the complete game.

### Includes

* All locations
* All chapters
* All NPCs
* Full dialogue
* All puzzles
* Complete evidence board
* Complete story

### Deliverable

The entire game is playable from beginning to end.

---

# Milestone 6 — Polish

## Goal

Improve overall quality.

### Tasks

* Bug fixing
* Performance optimization
* Lighting improvements
* Audio improvements
* UI polish
* Animation polish
* Visual polish
* Accessibility review

### Deliverable

A release candidate with production quality.

---

# Milestone 7 — Release

## Goal

Prepare the final version.

### Tasks

* Final QA
* Final balancing
* Build verification
* Backup project
* Create release build
* Prepare store assets
* Publish

### Deliverable

ROOTS Version 1.0

---

# Milestone Completion Rules

A milestone is complete only if:

* Every required task is finished.
* All critical bugs are fixed.
* Performance targets are met.
* Documentation is updated.
* The build is stable.

---

# Final Rule

Never skip milestones.

A strong foundation reduces development time, minimizes technical debt, and keeps ROOTS organized from the first prototype to the final release.
