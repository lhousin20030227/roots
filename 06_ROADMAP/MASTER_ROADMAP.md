# ROOTS — MASTER ROADMAP

Version: 1.0

Status: Active

---

## Purpose

This roadmap defines the complete development lifecycle of ROOTS, from the first concept to post-release support. Every AI agent and contributor is expected to follow it.

---

## Phase 1 — Pre-Production

### Goals

- Finalize the game vision.
- Complete the Master Bible.
- Finalize the story outline.
- Define gameplay systems.
- Define the technical architecture.
- Define art direction.
- Define audio direction.
- Create project documentation.

### Deliverables

- Master Bible
- Game Design Document
- AI Agent System
- Project Rules
- Standards
- Roadmaps

### Exit Condition

Project scope is frozen and approved.

---

## Phase 2 — Prototype

### Goals

Build a playable technical prototype.

### Systems

- Player Controller
- Camera
- Interaction
- Inventory
- Save/Load
- Pause Menu
- Settings
- Dialogue
- Evidence System

### Exit Condition

All core systems work together.

---

## Phase 3 — Production

### Goals

Create all game content.

### Content

- Levels
- Characters
- Props
- Documents
- Photos
- Audio Logs
- UI
- Story
- Puzzles
- Lighting

### Exit Condition

The entire game is playable.

---

## Phase 4 — Alpha

### Goals

- Complete feature implementation.
- Replace placeholders.
- Integrate all assets.
- Fix major issues.

### Exit Condition

The game is playable from beginning to end.

---

## Phase 5 — Beta

### Goals

- Bug fixing.
- Optimization.
- Polish.
- Balancing.
- QA.

### Exit Condition

A release candidate is approved.

---

## Phase 6 — Release

### Goals

- Publish ROOTS.
- Prepare launch assets.
- Monitor launch quality.
- Fix critical release issues.

### Exit Condition

Version 1.0 is released.

---

## Phase 7 — Post Release

### Goals

- Bug fixes.
- Performance improvements.
- Quality-of-life updates.
- Community feedback.
- Future planning.

---

## Development Priorities

1. Project stability.
2. Gameplay.
3. Story.
4. Art.
5. Audio.
6. Polish.

---

## Milestones

- M1: Documentation complete.
- M2: Prototype complete.
- M3: First playable.
- M4: Content complete.
- M5: Alpha approved.
- M6: Beta approved.
- M7: Release candidate.
- M8: ROOTS v1.0 released.

---

## Success Metrics

- Stable 60 FPS.
- No critical bugs.
- Story fully playable.
- Complete documentation.
- Consistent art style.
- Optimized build.
- QA approved.
- Producer approved.

---

## Project Rules

- Never expand scope without approval.
- Never skip QA.
- Never merge unfinished work.
- Always document completed tasks.
- Always follow project standards.

---

## Master Rule

Every task, feature, and decision must move ROOTS one step closer to a stable, complete, and release-ready game. If a task does not directly contribute to that goal, it should not be part of the project.
