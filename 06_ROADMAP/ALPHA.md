# ROOTS — ALPHA ROADMAP

Version: 1.0

---

# PURPOSE

The Alpha phase begins when all core game systems are implemented.

The objective is to create a fully playable game from start to finish, even if content is incomplete.

Focus on functionality, integration and stability.

---

# PRIMARY GOALS

Complete all planned gameplay systems.

Integrate all game locations.

Implement the complete story.

Replace temporary assets with production assets whenever possible.

Ensure the game is fully playable.

---

# ALPHA CHECKLIST

## Gameplay

* All core mechanics complete.
* Player progression functional.
* Inventory complete.
* Evidence system complete.
* Dialogue system complete.
* Save/Load functional.
* Settings functional.

---

## Story

* Main story playable.
* All chapters implemented.
* Dialogues integrated.
* Documents integrated.
* Audio logs integrated.

---

## Levels

* Every location playable.
* Objectives complete.
* Evidence correctly placed.
* Puzzles implemented.
* Navigation verified.

---

## Art

* Most placeholder assets removed.
* Character models integrated.
* Environment assets integrated.
* Pixel textures completed.
* Lighting baked.

---

## Audio

* Ambient sounds integrated.
* SFX complete.
* Music implemented.
* Voice recordings integrated (if available).
* Audio balancing started.

---

## UI

* Main Menu complete.
* Inventory complete.
* Evidence Board complete.
* Settings complete.
* Save/Load UI complete.

---

## Performance

* Stable FPS on target hardware.
* Memory usage acceptable.
* Loading times reasonable.
* Major bottlenecks identified.

---

## Documentation

* Technical documentation updated.
* Story documentation updated.
* Task documentation complete.
* Asset documentation updated.

---

# QA REQUIREMENTS

Every system tested.

Critical bugs documented.

Regression testing begins.

Gameplay flow verified.

Story consistency verified.

---

# EXIT CRITERIA

The project may leave Alpha only if:

The game is playable from beginning to end.

No missing core systems remain.

No blocker bugs exist.

All major content is implemented.

QA approves Alpha completion.

Producer approves transition to Beta.

---

# DELIVERABLES

Playable full game.

Updated documentation.

Bug reports.

Performance report.

QA report.

Asset completion report.

---

# FORBIDDEN

Adding new gameplay systems.

Changing project scope.

Major story rewrites.

Large architectural changes.

Replacing completed systems without approval.

---

# SUCCESS CONDITION

ROOTS is fully playable from start to finish with all planned features implemented. Remaining work is limited to polishing, bug fixing, optimization and content refinement before entering Beta.
