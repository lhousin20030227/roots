# UI ASSETS

## Purpose

This document defines the standards for all User Interface (UI) assets used in ROOTS.

UI assets should be clean, readable, consistent, and support the player's investigation without breaking immersion.

---

# Philosophy

The interface should help the player—not distract them.

UI assets should be minimal, modern, and easy to understand.

---

# UI Asset Categories

## Icons

Examples:

* Inventory
* Evidence
* Journal
* Settings
* Save
* Load
* Exit
* Search

Icons should remain simple and recognizable.

---

## Buttons

Examples:

* Primary Button
* Secondary Button
* Back
* Confirm
* Cancel
* Continue

Buttons should clearly communicate their function.

---

## Panels

Examples:

* Inventory Panel
* Evidence Board
* Journal
* Pause Menu
* Settings Menu
* Dialogue Window

Panels should maximize readability while keeping the world visible when appropriate.

---

## Cursors

Examples:

* Default Cursor
* Interaction Cursor
* Inspect Cursor
* Drag Cursor

Cursor changes should provide immediate feedback.

---

## Indicators

Examples:

* Loading Indicator
* Autosave Indicator
* Objective Updated
* New Evidence
* Interaction Prompt

Indicators should be subtle and short-lived.

---

# Visual Style

UI assets should be:

* Minimal
* Modern
* Clean
* Consistent
* High Contrast

Avoid excessive decoration.

---

# Colors

Use a restrained color palette.

Colors should communicate meaning.

Examples:

* White → Normal
* Blue → Information
* Green → Success
* Yellow → Warning
* Red → Error

Avoid overly saturated colors.

---

# Resolution

UI assets should scale correctly across different resolutions and aspect ratios.

They must remain sharp on:

* 1080p
* 1440p
* 4K
* Ultrawide displays

---

# File Formats

Preferred formats:

* `.svg` → Icons
* `.png` → UI graphics with transparency
* `.webp` → Optimized UI images where appropriate

---

# Optimization

* Keep file sizes small.
* Reuse shared assets.
* Use vector graphics where possible.
* Remove unused UI elements.

---

# Design Rules

* Consistent style.
* Clear readability.
* Minimal visual clutter.
* Fast loading.
* Accessible for all players.

---

# Final Rule

Every UI asset in ROOTS should quietly support the player. The interface should always feel like a natural extension of the investigation, never something that competes with the world itself.
