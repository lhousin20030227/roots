# ASSET RULES

## Purpose

This document defines the universal rules that every asset in ROOTS must follow before being included in the project.

These rules ensure consistency, realism, optimization, and long-term maintainability.

---

# Core Philosophy

Every asset must satisfy at least one of the following:

* Supports gameplay.
* Supports storytelling.
* Improves world building.
* Improves immersion.
* Improves visual quality.

If an asset serves none of these purposes, it should not exist.

---

# Realism

Assets should feel believable.

Avoid:

* Unrealistic proportions.
* Overly stylized designs.
* Cartoon-like appearance.
* Fantasy elements.
* Sci-fi technology.

ROOTS takes place in a realistic modern world.

---

# Quality Standards

Every asset must be:

* Clean
* Well-scaled
* Properly named
* Optimized
* Free of visual errors

Broken or unfinished assets are never accepted.

---

# Consistency

Assets must match the project's visual style.

Maintain consistency in:

* Scale
* Materials
* Colors
* Level of detail
* Wear and aging

Objects from the same environment should look like they belong together.

---

# Optimization

Optimization is mandatory.

Requirements:

* Remove hidden geometry.
* Use appropriate polygon counts.
* Reuse materials whenever possible.
* Compress textures.
* Minimize draw calls.

Visual quality must never unnecessarily reduce performance.

---

# File Integrity

Assets must:

* Use supported file formats.
* Contain no unused meshes.
* Contain no unused materials.
* Contain no duplicate textures.
* Have clean file structures.

---

# Naming

Every asset must follow the project's naming convention.

Names should be:

* Clear
* Consistent
* Descriptive
* Lowercase
* Underscore-separated

Example:

```text
hospital_bed_01.glb
family_photo_03.png
wooden_door_a.glb
```

---

# Reusability

Assets should be reusable whenever possible.

Examples:

* Chairs
* Tables
* Lamps
* Books
* Trees
* Road signs

Avoid creating duplicate assets with identical functionality.

---

# Collision

Interactive objects must include appropriate collision.

Collision should:

* Match the object's shape.
* Be lightweight.
* Prevent unrealistic movement.

---

# Story Assets

Important story assets require additional care.

Examples:

* Adoption papers.
* Family photographs.
* Wedding ring.
* Medical records.
* Diaries.

These assets should receive the highest visual quality because players inspect them closely.

---

# Forbidden Assets

Do not include:

* Placeholder models.
* Watermarked assets.
* Copyrighted content without permission.
* Duplicate assets.
* Unused files.
* Joke assets.
* Inconsistent art styles.

---

# Final Rule

Every asset in ROOTS should feel like a natural part of the world. Players should never notice the asset itself—they should only notice the believable world it helps create.
