# ICONS

## Purpose

This document defines the standards for all icons used in ROOTS.

Icons should communicate information instantly while maintaining a clean and consistent visual language.

---

# Philosophy

Icons should be:

* Simple
* Minimal
* Recognizable
* Consistent

Players should understand an icon without reading text whenever possible.

# Style

Icons should use:

* Simple geometric shapes.
* Clear silhouettes.
* Consistent stroke thickness.
* Minimal detail.

Avoid realistic or overly decorative icons.

---

# Size Standards

Base icon sizes:

* 16×16
* 24×24
* 32×32
* 48×48
* 64×64

Icons should scale cleanly without losing readability.

---

# Colors

Default:

* White

State colors:

* Blue → Information
* Green → Success
* Yellow → Warning
* Red → Error

Color should complement, not replace, shape recognition.

---

# File Format

Preferred formats:

* `.svg` → Master icons
* `.png` → Exported game-ready icons

SVG files should be the source of truth.

---

# Naming Convention

Examples:

```text
icon_inventory.svg
icon_evidence.svg
icon_save.svg
icon_settings.svg
icon_objective_completed.svg
```

Use lowercase letters and underscores only.

---

# Optimization

* Remove unused paths.
* Keep vector shapes clean.
* Avoid unnecessary anchor points.
* Export optimized PNG files when needed.

---

# Design Rules

* One meaning per icon.
* One consistent visual style.
* No text inside icons.
* No unnecessary decoration.
* High readability at small sizes.

---

# Final Rule

Every icon in ROOTS should communicate its purpose instantly. Players should recognize an action or system with a quick glance, allowing the interface to remain fast, intuitive, and unobtrusive.
