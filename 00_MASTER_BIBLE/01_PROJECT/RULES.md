# PROJECT RULES

## Purpose

These rules are mandatory for every AI Agent, developer, and future contributor. No task may violate these rules.

---

# Core Rules

1. Story always comes before features.
2. Gameplay must support the narrative.
3. Every new mechanic must have a clear purpose.
4. Simplicity is preferred over unnecessary complexity.
5. Consistency is more important than quantity.

---

# Forbidden Features

The following features must never be added unless explicitly approved by the Producer:

* Combat
* Weapons
* Monsters
* Zombies
* Supernatural powers
* Magic
* Ghosts
* Demons
* Multiplayer
* Battle Pass
* RPG statistics
* Skill Trees
* Loot systems
* Crafting systems
* Survival mechanics
* Random generated story events
* Arcade-style gameplay

---

# Story Rules

* Every clue must be logical.
* Every document must have a believable origin.
* Every location must contribute to the story.
* No plot holes.
* No forced drama.
* No information should appear without evidence.

---

# Gameplay Rules

* Exploration is the primary gameplay.
* Puzzles must be logical, not frustrating.
* Objectives should naturally guide the player.
* Every collected item must have a purpose.
* Backtracking should be meaningful.

---

# Art Rules

* Low Poly only.
* Pixel Art textures only.
* Consistent color palette.
* Realistic proportions.
* No exaggerated fantasy designs.

---

# Audio Rules

* Ambient sound is essential.
* Silence should be used intentionally.
* Music must never overpower exploration.
* Every important interaction should have audio feedback.

---

# Technical Rules

* Optimize from the beginning.
* Reusable systems only.
* Modular architecture.
* Clear file organization.
* Consistent naming conventions.

---

# AI Rules

Every AI Agent must:

* Read the MASTER_BIBLE before starting.
* Follow PROJECT_RULES without exception.
* Modify only assigned files.
* Never invent story details that contradict existing documents.
* Ask for clarification instead of making assumptions.

---

# Approval Rule

No major change to the story, gameplay, art direction, or project scope is considered official until approved by the Producer.
