EVIDENCE BOARD INTERFACE
Purpose
The Evidence Board Interface defines how players organize, view, and analyze collected evidence, transforming scattered discoveries into a coherent investigation tool that supports deduction without dictating conclusions.

Evidence Board Philosophy
The Evidence Board is the player's investigative workspace — a place to organize clues, make connections, and piece together the truth.
It should empower the player to think and deduce, never spoon-feed answers.

Evidence Board Access
Opening the Board

Opened via dedicated key (default: [B] on keyboard, [L1/LB] on controller).
Accessible from gameplay or pause menu.
Quick access from HUD (doesn't require pausing).
Smooth transition animation (fade in/out, not jarring).


Closing the Board

Closed via same key, ESC, or Back button.
Closes smoothly to resume gameplay.

Evidence Display Components
Evidence List
Shows all collected evidence organized by category:

Evidence Name — clear, descriptive title.
Evidence Type Icon — visual indicator (document, photo, audio, etc.).
Brief Description — one-line context (where found, what it shows).
Status Badges — [NEW], [UNREAD], [IMPORTANT] (optional tags).
Timestamp — when evidence was collected.


Evidence Details Panel
When evidence is selected:

Full Title & Description — complete information.
Evidence Content — full text of document, zoomed photo, audio player, etc.
Related Evidence — links to connected clues (optional).
Player Notes — any notes player added about this evidence.


Timeline View
Chronological display of all evidence ordered by in-game date:
┌─────────────────┐
│ 1995            │
│ └─ Photo        │
│                 │
│ 1998            │
│ ├─ Med Record   │
│ └─ Letter       │
│                 │
│ 2010            │
│ └─ Cassette     │
│                 │
│ Unknown Date    │
│ └─ Other Items  │
└─────────────────┘

Helps player understand chronological progression of evidence.
Gaps in timeline reveal what's still unknown.
Missing dates kept hidden until discovered.


Filters
Allow player to focus on specific types of evidence:

All — show everything collected.
New — show only recently collected (highlighted as [NEW]).
Character — evidence related to specific people.
Location — evidence found at specific locations.
Story Evidence — evidence directly tied to main narrative.
Character Evidence — evidence about specific characters.
Location Evidence — evidence describing places.
Unread — documents not yet read.


Evidence Categories
Documents

Letters
Official records
Handwritten notes
Books or journals

Display: Full text in readable font, page images if scanned.

Photographs

Family photos
Personal snapshots
Official photographs

Display: Zoomable image, description, date if known.

Audio Evidence

Voice recordings
Voicemail messages
Cassette tapes
Audio logs

Display: Audio player with play/pause/seek controls, transcript (optional).

Physical Items

Personal belongings
Objects found at locations
Puzzle pieces

Display: 3D model or photo, description, where found.

Player Notes System
Players may add personal notes to the Evidence Board:
"Father's expression in photo looks sad.
Why would he be unhappy in a family photo?"

Completely optional (never required for progression).
Notes saved with evidence.
Multiple notes possible per piece of evidence.
Notes deleted by player if desired.


Evidence Connections
Optional system to show relationships between evidence:
Father's Photo ──→ Medical Record
    ↓                    ↓
    Mother's Letter ←────┘

Player can view how pieces of evidence relate.
Connections revealed only after collecting related evidence (no spoilers).
Never mandatory (visual option, player can ignore).


Evidence Organization
Manual Organization

Players may manually reorganize evidence lists (drag-to-reorder).
Custom groupings possible (create personal categories).


Automatic Organization

Evidence organized by collection order (chronological).
Evidence organized by category (type).
Evidence organized by timeline (in-game date).

Player chooses preferred organization method.

Evidence Status Indicators

[NEW] — evidence collected this session.
[UNREAD] — document not yet read.
[IMPORTANT] — evidence critical to investigation (optional system marking).
[VERIFIED] — evidence confirmed through investigation.


Accessibility Features

Large, readable text (adjustable font size in settings).
Audio transcripts provided for audio evidence (if available).
High contrast text and backgrounds.
Colorblind-friendly icons and indicators.
Keyboard navigation (arrow keys, Enter to select, ESC to close).
Screen reader support (evidence descriptions read aloud if needed).


Performance Rules

Evidence Board loads instantly (no lag when opening).
Image previews load smoothly (no stuttering when browsing).
Audio playback smooth and reliable.
No frame rate impact from Board being open.


Forbidden Evidence Board Practices
Do not include:

Automatic evidence highlighting or "guiding" solutions.
Forced evidence organization (player choice priority).
Spoiler text or premature reveals.
Confusing interface or navigation.
Evidence that disappears or becomes unavailable.
No way to reread or replay collected evidence.


Final Rule
The Evidence Board should feel like the player's investigative companion — a place to think, organize, remember, and piece together the truth at their own pace, never rushing or dictating conclusions.
