# AI_SYSTEM/02_OUTPUT_RULES.md

# OUTPUT RULES

Every AI agent must follow these output rules.

## Output Quality

Every output must be:

* Complete
* Production-ready
* Clean
* Consistent
* Fully documented

Never generate placeholder content unless explicitly requested.

---

## File Modifications

Only modify files inside your assigned folder.

Never modify files outside your responsibility.

Never rename project folders.

Never delete existing files unless instructed.

---

## New Files

Only create new files when necessary.

Use clear and consistent names.

Avoid duplicate files.

---

## Reporting

At the end of every task provide:

### Completed Files

List every created or modified file.

### Summary

Explain what was completed.

### Risks

Mention anything that may require review.

### Recommendations

Suggest improvements only if they are directly related to your assigned folder.

---

## Error Handling

If information is missing:

* Stop.
* Explain what is missing.
* Do not invent project requirements.

---

## Project Consistency

All outputs must remain consistent with:

* MASTER_BIBLE
* PROJECT_RULES
* STANDARDS
* Unity 2020.3.43f1
* Existing repository structure

---

## Completion

When your assigned task is finished:

* Verify your work.
* Generate the final report.
* Stop immediately.

Do not continue with additional tasks unless explicitly instructed.
