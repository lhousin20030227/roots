## Review Checklist

### Functionality

- Feature works as designed.
- All interactions respond correctly.
- No broken triggers.
- No missing functionality.
- Matches Gameplay Design.

PASS / FAIL

### Bugs

- No critical bugs.
- No game-breaking bugs.
- No visual bugs.
- No audio bugs.
- No UI bugs.

PASS / FAIL

### Story Consistency

- No contradictions with Story Bible.
- No contradictions with Lore Bible.
- Dialogue triggers correctly.
- Evidence appears in correct order.
- No missing story flags.

PASS / FAIL

### Save / Load

- Save works correctly.
- Load restores correct state.
- Inventory persists correctly.
- Objectives persist correctly.
- No data loss.

PASS / FAIL

### Stability

- No crashes.
- No freezes.
- No memory leaks during testing.
- Stable across multiple play sessions.
- Stable across scene transitions.

PASS / FAIL

### Regression

- Previously fixed bugs do not reappear.
- New changes did not break existing systems.
- Core loop still functions correctly.

PASS / FAIL

### Reproducibility

- Bug steps are clear.
- Bug is reproducible.
- Conditions are documented.
- Severity is justified.

PASS / FAIL

### Compatibility

- Works on target hardware.
- Works at intended resolution.
- Works with intended input devices.
- No platform-specific issues.

PASS / FAIL
